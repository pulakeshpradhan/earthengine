# আর্থ ইঞ্জিনে স্কেল ও প্রজেকশন (বাংলা সারসংক্ষেপ)

> **নোট:** এই ফাইলটি `scale_projection.md` (৮২KB) এর বাংলা সারসংক্ষেপ। সম্পূর্ণ বিস্তারিত বিবরণের জন্য মূল ইংরেজি ফাইল দেখুন।

---

## মূল ধারণাসমূহ

### ১. প্রজেকশন কী?

প্রজেকশন হলো ত্রিমাত্রিক পৃথিবীকে দ্বিমাত্রিক মানচিত্রে উপস্থাপনের পদ্ধতি। প্রতিটি প্রজেকশন কিছু বৈশিষ্ট্য সংরক্ষণ করে এবং অন্যগুলো বিকৃত করে।

### ২. আর্থ ইঞ্জিনের অনন্য পদ্ধতি: "Pull" ভিত্তি

- আর্থ ইঞ্জিন অপারেশনগুলো ডিফল্টভাবে **ক্লায়েন্ট/আউটপুট প্রজেকশনে** চলে।
- আপনি `Map.addLayer()` কল করলে, আর্থ ইঞ্জিন চাহিদা মতো পিক্সেল গণনা করে।
- এটি "lazy evaluation" — গণনা তখনই হয় যখন প্রয়োজন।

### ৩. ইমেজ পিরামিড

- প্রতিটি ইমেজের একাধিক রেজোলিউশন লেভেল আছে।
- জুম আউট করলে নিম্নতর রেজোলিউশন ব্যবহৃত হয়।
- জুম ইন করলে উচ্চতর রেজোলিউশন ব্যবহৃত হয়।

### ৪. স্কেল কীভাবে বিশ্লেষণ প্রভাবিত করে

- `reduceRegion()` বা `Export` এ `scale` প্যারামিটার পিক্সেল আকার নির্ধারণ করে।
- ভুল স্কেল ব্যবহারে ফলাফল সম্পূর্ণ ভিন্ন হতে পারে।

### ৫. কখন প্রজেকশন ম্যানুয়ালি নির্দিষ্ট করবেন

- **বেশিরভাগ ক্ষেত্রে**, আর্থ ইঞ্জিন নিজেই সামলায়।
- `reproject()` শুধুমাত্র তখনই ব্যবহার করুন যখন:
  - কার্নেল অপারেশনে নির্দিষ্ট স্কেল লাগে (যেমন- slope)
  - বিভিন্ন রেজোলিউশনের ডেটা মিশানো হচ্ছে

## সাধারণ কোড প্যাটার্ন

### প্রজেকশন তথ্য পরীক্ষা

```javascript
var image = ee.Image('LANDSAT/LC08/C02/T1_TOA/LC08_044034_20140318').select(0);
print('প্রজেকশন:', image.projection());
print('মিটারে স্কেল:', image.projection().nominalScale());
```

### reduceRegion-এ স্কেল নির্দিষ্ট করা

```javascript
var stats = image.reduceRegion({
  reducer: ee.Reducer.mean(),
  geometry: roi,
  scale: 30,  // মিটারে পিক্সেল আকার
  maxPixels: 1e9
});
```

### reproject() ব্যবহার (সতর্কতার সাথে)

```javascript
var reprojected = image.reproject({
  crs: 'EPSG:4326',
  scale: 30
});
```

---

## ⚠️ সতর্কতা

- `reproject()` অনেক বেশি ডেটা অনুরোধ করতে পারে
- সম্ভব হলে `scale` প্যারামিটার ব্যবহার করুন `reproject()` এর পরিবর্তে
- ভিন্ন প্রজেকশনের ইমেজ মিশালে ডিফল্ট আউটপুট WGS84 (EPSG:4326) হয়
